# Модель базы данных

Версия: 0.1

Статус: В разработке

---

# 1. Назначение

База данных является ядром Региональной платформы пространственной аналитики недвижимости.

В качестве СУБД используется PostgreSQL 16+ с расширением PostGIS.

Основные требования:

- централизованное хранение пространственных данных;
- поддержка версионности;
- высокая скорость пространственных запросов;
- поддержка ETL;
- поддержка аналитических расчетов.

---

# 2. Общая структура

Используется разделение на схемы PostgreSQL.

```

```
raw
staging
core
market
factors
analytics
report
system
```

---

# 3. Назначение схем

## raw

Исходные данные.

Правила:

- данные не изменяются;
- допускается только загрузка новых версий;
- запрещено ручное редактирование.

Примеры таблиц

```
raw.egrn_land_parcel
raw.egrn_building
raw.deals
raw.offers
```

---

## staging

Промежуточная обработка.

Используется для

- очистки;
- нормализации;
- проверки качества.

После завершения обработки данные могут удаляться.

---

## core

Главная схема платформы.

Содержит эталонные сведения.

Все приложения используют исключительно эту схему.

---

## market

Рыночные наблюдения.

Включает

- сделки;
- объявления;
- аренду;
- историю изменения цен.

---

## factors

Хранение рассчитанных факторов стоимости.

Каждый фактор имеет собственный код.

---

## analytics

Результаты аналитических расчетов.

Например

- рыночные зоны;
- кластеры;
- индексы;
- модели.

---

## report

Представления.

Используются

- отчетами;
- API;
- QGIS.

---

## system

Служебная информация.

Например

- журнал загрузок;
- версии;
- пользователи;
- настройки.

---

# 4. Основные сущности

На первом этапе создаются следующие сущности.

| Сущность | Схема |
|----------|--------|
| object | core |
| land_parcel | core |
| building | core |
| address | core |
| market_observation | market |
| factor | factors |
| factor_value | factors |
| event | system |
| dataset | system |

---

# 5. Пространственные требования

Все пространственные таблицы используют

Geometry

SRID = 3857

При необходимости допускается хранение геометрии в 4326.

Во всех пространственных таблицах создается индекс GiST.

---

# 6. Версионность

Удаление записей запрещено.

Используются поля

created_at

updated_at

valid_from

valid_to

is_actual

---

# 7. Первоочередные задачи

Первая версия включает

- объекты недвижимости;
- адреса;
- рыночные наблюдения;
- пространственные факторы;
- цифровой паспорт объекта.

После реализации данных сущностей начинается разработка ETL.
